# 🐛 Prompt 调试 Checklist

> Prompt 效果不理想,按顺序排查。

---

## 🚦 诊断流程图

```
Prompt 输出有问题
    ↓
是"格式/类型错误"? → 见【格式问题】
是"内容错/瞎编"? → 见【幻觉问题】
是"理解错任务"? → 见【指令问题】
是"不稳定/每次不同"? → 见【稳定性问题】
是"太长/太短/太水"? → 见【长度与质量问题】
是"拒答/过度谨慎"? → 见【过度对齐问题】
```

---

## 🧱 格式问题

**症状**:JSON 解析失败、字段缺失、多了 Markdown 包裹

| 排查 | 修复 |
|---|---|
| 输出被 ` ```json ` 包起来了? | 加一句 `"只输出 JSON,不要 Markdown 代码块包裹"` |
| 字段名和 schema 不一致? | Schema 示例写得更死:`{"name": "string"}` 而不是 `{name: str}` |
| 解析前后有多余文字? | 用 `<output>` 标签包裹,后处理取标签内容 |
| 偶尔出错(<5%)? | 加解析失败的 retry,错误信息回灌 |
| 经常出错(>10%)? | 改用 API 层的 Structured Outputs / Tool Use |

---

## 👻 幻觉问题

**症状**:模型编造事实、字段有值但原文没提

| 排查 | 修复 |
|---|---|
| 有没有说"原文没有就填 null"? | 加 `"未提及的字段一律填 null,不要推测"` |
| Few-shot 例子里有"空字段"案例吗? | 补一个字段缺失的示例 |
| Temperature 太高? | 降到 0.0-0.3 |
| 任务是"总结"还是"提取"? | 提取型任务明确写"只从原文"不要"总结原文" |
| 资料太长/信息埋太深? | 检查是否"中间迷失",把关键信息提前 |
| 给了矛盾的 Few-shot? | 检查示例内部的一致性 |

---

## 🎯 指令问题

**症状**:模型没做你想让它做的事、做多了 / 做少了

| 排查 | 修复 |
|---|---|
| `<task>` 是不是写的太泛? | 从"帮我分析这份合同"改为"提取以下 5 个字段" |
| Role 过于笼统? | "专家" → "10 年经验的电气采购经理" |
| 指令里有矛盾? | 检查 `<rules>` 是否自相冲突 |
| 没有例子? | 加 1-3 个 Few-shot |
| 例子质量差? | 每个例子都要展示一个关键点,不要重复 |

---

## 🎲 稳定性问题

**症状**:同一输入,今天输出 A,明天输出 B

| 排查 | 修复 |
|---|---|
| Temperature 是多少? | 稳定类任务(提取/分类)设 0.0-0.2 |
| 输出格式太"自由"? | 用 Structured Outputs 锁死结构 |
| 输入有随机性? | 检查是否每次拼接顺序不同 |
| 模型本身波动大? | 用 Self-Consistency(跑 N 次投票) |
| 换一个版本试试 | 从 Haiku 升到 Sonnet,或反之 |

---

## 📏 长度与质量问题

**症状**:太长/太短/太水

| 症状 | 修复 |
|---|---|
| 太啰嗦 | `"不要开场白,直接给答案,字数 ≤ N"` |
| 太简短 | 让模型先输出 `<analysis>`,再给 `<answer>` |
| 内容空洞 | 换更具体的 Role,加约束"结论必须引用原文" |
| 重复废话("总的来说...") | 明确禁:`"不要总结、重复、开场白"` |
| 没有洞察 | 加"你的读者会觉得'哦原来如此'的点是什么?" |

---

## 🚫 过度对齐问题(过度谨慎)

**症状**:模型拒答正当请求,或加太多"请谨慎""仅供参考"

| 排查 | 修复 |
|---|---|
| 请求被模型误判为"不安全"? | 在 Prompt 中说明合法用途和场景 |
| 输出被一堆免责声明淹没? | 加 `"不要添加无关的免责声明,除非涉及投资/医疗"` |
| 模型总说"我建议咨询专业人士"? | 给更具体的 Role + 授权("你是授权给出 X 的专家") |

## 🧪 排查工具箱

```python
# 1. 比对 A/B 版本
def compare_versions(prompt_a, prompt_b, inputs):
    for inp in inputs:
        a = call_llm(prompt_a, inp)
        b = call_llm(prompt_b, inp)
        print(f"Input: {inp}\nA: {a}\nB: {b}\n---")

# 2. 跑 N 次看稳定性
def check_stability(prompt, input_data, n=5):
    outputs = [call_llm(prompt, input_data) for _ in range(n)]
    return outputs  # 肉眼看差异

# 3. Temperature 扫描
for t in [0.0, 0.3, 0.7, 1.0]:
    print(f"T={t}: {call_llm(prompt, input, temperature=t)}")
```

## 💡 黄金法则

**先假设是 Prompt 问题,不是模型能力问题。**

90% 的"效果不好"都是 Prompt 写得不够清楚,而不是模型太笨。换模型之前先把 Prompt 调好。

**改 Prompt 的顺序:**
1. 先加 Few-shot
2. 再加 CoT
3. 再调 Role 具体度
4. 最后才考虑换模型
